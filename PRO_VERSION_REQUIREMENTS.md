# Metis 专业版功能需求文档

## 1. 目标

专业版不是单纯卖更多按钮，而是把「一次性排盘工具」升级成「可持续使用的命理分析工作台」。

第一阶段目标：

- 让用户清楚知道免费版和专业版差异。
- 先实现专业版权益展示、解锁逻辑、使用次数限制、报告下载、AI 追问额度。
- 支持后续接入登录、支付、云端同步和移动端 App。

一句话定位：

> 免费版让用户看懂一个基础命盘，专业版让用户持续深度分析自己和重要关系。

## 2. 用户

### 2.1 免费体验用户

用户刚进入产品，只想快速排盘、看看准不准。

产品策略：

- 免费提供基础排盘和核心解读。
- 在用户看见价值后，再展示专业版入口。
- 不强行打断核心体验。

### 2.2 专业版潜在用户

用户已经对结果感兴趣，想看更多维度、更多时间周期、更多 AI 追问。

产品策略：

- 明确展示可解锁内容。
- 让用户知道升级后马上能得到什么。
- 优先强调「深度报告」「更多解读」「AI 追问」「隐私与自有模型」。

### 2.3 高信任用户

用户重视隐私、希望使用自己的模型 API Key，或者未来会多端同步。

产品策略：

- 支持自有模型 API Key。
- AI 对话数据默认本地化。
- 未来支持移动端同步时，明确同步范围和隐私说明。

## 3. 专业版权益

### 3.1 解读项目解锁

免费版默认开放：

- 命格总览
- 感情
- 事业
- 财运
- 健康
- 性格

专业版新增解锁 7 项：

- 兄弟
- 子女
- 迁移
- 人际
- 田宅
- 福德
- 父母

验收标准：

- 未开通时，新增 7 项显示为锁定状态。
- 点击锁定项时，弹出专业版权益说明。
- 开通后，新增 7 项可以正常生成 AI 解读。
- 每个项目都使用统一输出结构：一句话结论、命盘依据、优势、风险、现实建议、未来一年重点。

### 3.2 排盘范围解锁

免费版默认开放：

- 本命盘

专业版开放：

- 本命
- 大限
- 流年
- 流月
- 流日
- 时辰

验收标准：

- 排盘页增加「排盘类型」切换。
- 免费用户只能选择本命。
- 专业版用户可以切换大限、流年、流月、流日、时辰。
- 切换后命盘、摘要和 AI 解读同步更新。
- 如果算法暂未完整支持某个类型，先显示「即将开放」而不是假结果。

### 3.3 AI 实时问答

免费版：

- 每日 AI 追问额度较少，例如 3 次。

专业版：

- 平台 AI 每日 20 次。
- 接入自有模型后不限次数，受用户自己的 API Key 额度限制。

验收标准：

- AI 问答区显示今日剩余额度。
- 达到免费额度后，引导升级专业版。
- 专业版达到平台 AI 20 次后，引导接入自有模型。
- 自有模型模式下，产品不再限制追问次数。
- AI 失败时必须有清晰错误提示和本地兜底文案。

### 3.4 AI 对话本地化

默认规则：

- AI 对话记录优先保存在浏览器本地。
- 不登录时不上传云端。
- 用户主动同步或登录后，才进入云端同步流程。

验收标准：

- 页面明确提示「当前对话仅保存在本机」。
- 用户可以一键清空本地对话。
- 隐私说明页补充 AI 对话数据存储规则。

### 3.5 自有模型 API Key

专业版支持用户填写自己的模型 API Key。

第一阶段支持：

- OpenAI compatible endpoint
- API Key
- Model name

后续可扩展：

- OpenAI
- Anthropic
- Google Gemini
- DeepSeek
- 智谱

验收标准：

- API Key 只保存在本地浏览器。
- 输入框默认隐藏 Key 内容。
- 提供测试连接按钮。
- 连接失败时显示具体原因。
- 使用自有模型时，AI 请求走用户配置。
- 不把用户 API Key 发送到平台数据库。

### 3.6 三纪内容库

专业版逐步开放三纪内容：

天纪：

- 紫微斗数
- 易经
- 推命学
- 面相学
- 字理文化

地纪：

- 风水地理
- 形势
- 理气
- 阳宅阴宅

人纪：

- 针灸
- 黄帝内经
- 神农本草
- 伤寒金匮

第一阶段实现策略：

- 先做内容索引页和专业版锁定状态。
- 已有内容可直接开放阅读。
- 未完成内容不做假入口，只显示「整理中」。

验收标准：

- 三纪内容有统一入口。
- 专业版内容锁定状态清晰。
- 已开放内容可以搜索、阅读、收藏。
- 未开放内容不会让用户误以为已经可用。

### 3.7 紫微斗数全盘分析报告

专业版核心交付物：

- 生成完整全盘分析报告。
- 支持下载。
- 支持自定义姓名。

报告结构：

1. 基本信息
2. 命盘总览
3. 十二宫重点
4. 主星与格局
5. 四化分析
6. 大限趋势
7. 流年重点
8. 感情、事业、财运、健康专题
9. 风险提醒
10. 现实行动建议
11. 免责声明

第一阶段下载格式：

- 优先 Markdown 或 HTML。
- PDF 放到第二阶段，不为了排版拖慢 MVP。

验收标准：

- 用户可以输入报告姓名。
- 点击生成后得到完整报告。
- 报告可以复制。
- 报告可以下载为 `.md` 或 `.html`。
- 下载文件名包含姓名和生成日期。

### 3.8 古籍与批注

专业版权益：

- 六部古籍
- 四层精解全开
- 倪师批注

第一阶段实现策略：

- 先把「古籍库」分成免费阅读和专业精解。
- 免费用户可看原文或部分章节。
- 专业版解锁精解、批注、搜索增强。

验收标准：

- 古籍详情页展示原文、白话、精解、批注四层结构。
- 免费用户看到专业层级锁定提示。
- 专业版用户可完整展开。
- 搜索结果能区分原文和批注。

### 3.9 移动端 App 权益

专业版未来权益：

- 手机端 App 专业版优先开放体验。
- 数据同步。
- 优先 iOS，后续 Android。

第一阶段处理：

- 只做权益展示和等待名单。
- 不承诺具体上线日期。

验收标准：

- 专业版页面展示 App 优先体验权益。
- 用户可以选择「希望参与 iOS 测试」。
- 等待名单记录邮箱或微信。

## 4. 产品分层

### 4.1 免费版

- 基础排盘
- 本命盘
- 6 项基础解读
- 每日少量 AI 追问
- 基础古籍阅读
- 基础分享

### 4.2 专业版

- 13 项完整解读
- 本命、大限、流年、流月、流日、时辰
- 平台 AI 每日 20 次
- 自有模型不限次数
- AI 对话本地化
- 自有模型 API Key
- 三纪专业内容
- 全盘分析报告下载
- 古籍四层精解和批注
- 移动端 App 优先体验

### 4.3 后续高级版或单次购买

暂不在第一阶段做。

未来可考虑：

- 单份深度报告购买
- 年度运势报告
- 合盘深度报告
- 企业或咨询师版本

## 5. 页面需求

### 5.1 专业版页面 `/login` 或 `/pro`

建议把当前登录页改造成专业版页面，未来再独立出 `/login`。

页面结构：

1. 顶部价值说明
2. 免费版 vs 专业版对比
3. 专业版权益列表
4. 重点权益卡片：完整解读、全部排盘、AI 问答、全盘报告、自有模型
5. 等待名单或开通入口
6. 隐私说明
7. 常见问题

验收标准：

- 用户 5 秒内知道专业版能解锁什么。
- 不出现空按钮。
- 未接支付前，按钮文案使用「加入专业版等待名单」或「预约开通」。
- 表单至少收集邮箱或微信。

### 5.2 排盘结果页 `/chart`

新增：

- 专业版权益入口。
- 锁定解读项。
- 排盘类型切换。
- AI 额度展示。
- 报告生成入口。

验收标准：

- 免费用户仍能顺畅完成基础排盘。
- 专业版入口出现在用户看到结果之后。
- 锁定状态不影响基础功能。

### 5.3 AI 设置页或弹窗

新增：

- 自有模型配置。
- 本地保存说明。
- 测试连接。
- 清空配置。

验收标准：

- API Key 不明文展示。
- 用户可以随时删除本地 Key。
- 自有模型启用状态清晰。

### 5.4 报告页或报告弹窗

新增：

- 姓名输入。
- 报告预览。
- 复制。
- 下载。

验收标准：

- 生成过程有 loading。
- 失败时不丢失命盘数据。
- 下载内容完整可读。

## 6. 数据与状态

### 6.1 用户权益状态

第一阶段不做完整账号系统时，可以使用本地状态模拟：

```ts
type Plan = "free" | "pro";

type EntitlementState = {
  plan: Plan;
  aiDailyLimit: number;
  aiUsedToday: number;
  unlockedTopics: string[];
  enabledChartModes: string[];
  byokEnabled: boolean;
};
```

后续接入 Supabase 后迁移到服务端：

- users
- subscriptions
- usage_limits
- reports
- waitlist

### 6.2 本地存储

可本地保存：

- 最近命盘
- AI 对话
- 自有模型配置
- 报告草稿
- 等待名单提交记录

不应本地长期明文保存：

- 支付信息
- 身份证件
- 敏感个人资料

API Key 第一阶段可保存在 localStorage，但页面必须明确说明「仅保存在当前浏览器」。后续更安全方案是使用 Web Crypto 加密或只在当前会话保存。

## 7. 技术实现顺序

### Phase 1：需求闭环和转化页

- 新增专业版需求文档。
- 改造 `/login` 为专业版权益页。
- 增加等待名单表单，本地保存即可。
- 补隐私说明。

### Phase 2：前端解锁状态

- 增加权益配置文件。
- 排盘结果页增加锁定解读项。
- AI 区显示额度。
- 专业版入口从结果页跳转。

### Phase 3：报告下载

- 增加报告生成模块。
- 支持 Markdown 或 HTML 下载。
- 支持自定义姓名。

### Phase 4：BYOK 自有模型

- 增加模型配置弹窗。
- 支持 OpenAI compatible endpoint。
- 接入测试连接。
- AI 请求支持平台模型和自有模型切换。

### Phase 5：真实会员系统

- 接 Supabase Auth。
- 接 subscriptions 表。
- 接支付系统。
- 把本地权益迁移为服务端权益。

## 8. 不做范围

第一阶段暂不做：

- 真实支付
- 复杂会员后台
- iOS App
- Android App
- PDF 高级排版
- 所有三纪内容一次性补全
- 医疗、投资、法律类确定性建议

## 9. 风险

### 9.1 功能过多导致上线变慢

处理：

- 第一阶段只做权益页、锁定状态、等待名单。
- 真正复杂功能按报告、BYOK、会员系统逐步做。

### 9.2 命理内容涉及高风险建议

处理：

- 所有 AI 解读必须强调文化参考。
- 健康、投资、法律相关内容必须避免确定性建议。
- 页面底部保留免责声明。

### 9.3 用户不理解专业版价值

处理：

- 不只写功能名，要写具体收益。
- 例如「流年」解释为「看未来一年重点变化」。
- 「自有模型」解释为「使用你自己的 API Key，数据更私密，次数不受平台限制」。

## 10. 验收清单

- [ ] 专业版权益文案完整。
- [ ] 免费版和专业版差异清晰。
- [ ] `/login` 或 `/pro` 不再像占位页。
- [ ] 等待名单可以提交。
- [ ] 排盘结果页有专业版入口。
- [ ] 锁定项点击后能解释升级价值。
- [ ] AI 额度展示清晰。
- [ ] 报告下载 MVP 可用。
- [ ] 自有模型 API Key 不上传平台。
- [ ] 隐私说明包含 AI 对话和 API Key。
- [ ] 移动端展示不溢出。
- [ ] `npm run build` 通过。

## 11. MVP 决策

最小可上线版本只做 4 件事：

1. 专业版页面。
2. 等待名单。
3. 排盘结果页锁定专业权益入口。
4. 全盘报告 Markdown 下载。

这样可以最快验证：

- 用户是否愿意点专业版。
- 用户最想要哪个权益。
- 用户是否愿意留下联系方式。
- 深度报告是否有付费价值。
