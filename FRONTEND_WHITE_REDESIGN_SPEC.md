# Metis 白色主视觉前端修复文档

日期：2026-07-07

## 1. 修复目标

当前本地项目虽然可以运行，但页面前端效果和线上产品感不一致：内部页面仍保留开源 Demo 味道，视觉层级、字体、表单排版、页面逻辑都不够统一。

本轮修复目标：

- 整体以白色为主，不再使用深色、玄学感厚重背景作为主界面。
- 字体、间距、线条、表单控件向参考图靠拢，形成干净、克制、产品化的 Metis 风格。
- 起盘页、合盘页、结果页不再像 Demo，而是统一为正式产品页面。
- 页面逻辑和视觉状态完整：填写前、校验、加载、生成后、失败、追问都要有明确状态。
- 首页、起盘、合盘、功能入口页共用同一套视觉系统。

## 2. 参考图视觉拆解

参考图体现的是一个白色中轴表单页，关键特征如下：

- 背景：纯白或接近纯白，几乎没有装饰背景。
- 排版：内容居中，页面中轴非常明确。
- 顶部识别：
  - 小号英文编号：`01 / DESTINY ENGINE`
  - 大号中文标题：`起紫微命盘`
  - 小号说明：`输入出生年月日时`
  - 标题下方一条细灰分割线。
- 表单卡片：
  - 最大宽度约 432px。
  - 白底、浅灰边框、圆角 20px 左右。
  - 内边距约 28px。
  - 无大面积阴影，只保留极弱层次。
- 表单标题：
  - `—— 输入生辰八字 ——`
  - 字重较高，字距略宽。
  - 下方有阶段进度线，黑色表示当前/完成，浅灰表示未完成。
- 控件：
  - 输入框高度约 48px。
  - 圆角 12px。
  - 边框浅灰。
  - 文本黑色，placeholder 浅灰。
  - 下拉框保持原生识别，但样式统一。
- 字体气质：
  - 中文标题粗黑、醒目。
  - 表单文字中等字重，不浮夸。
  - 英文和小标签使用较大字距，但不使用负字距。

## 3. 设计原则

### 3.1 白色为主

页面主背景统一使用：

```css
--page-bg: #ffffff;
--panel-bg: #ffffff;
--soft-bg: #f7f7f6;
--field-bg: #fafafa;
```

禁止在主流程页面使用大面积深色、紫色、金色渐变、重阴影、厚装饰纹理。

### 3.2 黑白灰为骨架，少量暖色点缀

主色：

```css
--ink: #17181c;
--ink-soft: #4b5563;
--muted: #8a9099;
--line: #e5e7eb;
--line-strong: #111318;
--accent: #9a7a42;
```

暖色只用于极少量状态提示或品牌细节，不作为大面积背景。

### 3.3 中轴优先

起盘前、合盘前、登录/专业版提示页都采用中轴布局。

页面结构：

```text
顶部导航
小编号
大标题
副标题
细分割线
主表单卡片
底部说明/免责声明
```

结果页才切换为工作台布局：

```text
顶部导航
左侧/中心命盘
右侧解读
底部/侧边追问
```

## 4. 字体规范

### 4.1 字体栈

```css
font-family:
  Inter,
  "PingFang SC",
  "Hiragino Sans GB",
  "Microsoft YaHei",
  Arial,
  sans-serif;
```

不额外引入网络字体，避免本地不可用、加载抖动。

### 4.2 字号层级

桌面端：

```text
H1 页面主标题：56px / 1.05 / 800
页面编号：13px / 1.4 / 500 / letter-spacing 0.42em
副标题：14px / 1.6 / 400
表单标题：14px / 1.4 / 700 / letter-spacing 0.28em
字段 label：14px / 1.5 / 500
输入内容：15px / 1.4 / 500
说明文字：13px / 1.6 / 400
按钮文字：15px / 1.4 / 700
```

移动端：

```text
H1：40px / 1.08 / 800
表单卡片：宽度 calc(100vw - 32px)
字段 label：13px
输入内容：14px
```

### 4.3 标题规则

- 首页仍保留 `METIS` 品牌识别。
- 起盘页 H1 使用 `起紫微命盘`。
- 合盘页 H1 使用 `紫微合盘分析`。
- 其他入口页 H1 直接写功能名，不写营销文案。

## 5. 页面结构规范

### 5.1 全局壳

新增或重构一个白色产品壳：

```text
components/white/WhitePageShell.tsx
components/white/WhitePageHero.tsx
components/white/WhiteFormCard.tsx
components/white/WhiteField.tsx
components/white/WhiteSegmentedControl.tsx
components/white/WhitePrimaryButton.tsx
```

用途：

- 统一白底页面背景。
- 统一顶部导航。
- 统一中轴标题区。
- 统一表单卡片。
- 避免每个页面手写 inline style。

### 5.2 顶部导航

桌面端：

```text
METIS / 紫微斗数 · 三纪        起盘 · 合盘    中 / EN    专业版
```

规则：

- 白底透明度 0.96。
- 不使用大阴影。
- 当前页可以使用黑色字重区分。
- 分隔点使用浅灰 `·`。

移动端：

```text
METIS                  ☰
```

菜单展开后显示：起盘、合盘、知识库、古籍库、专业版。

### 6.2 表单逻辑

必须保留当前已有能力：

- 年月日选择。
- 时分选择。
- 省市联动。
- 真太阳时换算。
- 未知出生时间。
- 性别选择。

- 右侧解读卡片白底浅灰边框。
- 追问输入框固定在解读卡片底部或卡片下方。

## 11. 页面状态规范

每个核心页面必须有这些状态：

```text
idle       初始未填写
partial    已填写部分信息
invalid    表单校验失败
loading    正在生成
success    生成完成
error      生成失败
```

错误提示规则：

- 不弹浏览器 alert。
- 不用红色大块。
- 在字段下方用 12px 红字提示。
- 全局错误在按钮上方或卡片底部显示。

```text
npm.cmd run build
```


浏览器用户流：

## 15. 当前判断

推荐先从 `/chart` 开始修。

原因：

- 参考图就是起盘页。
- 起盘页是核心转化路径。
- `BirthForm` 是合盘页也会复用的组件。
- 先把起盘页修好，后面的合盘页会自然统一。

修复优先级：

```text
P0：白色设计 token + 起盘页表单视觉
P0：BirthForm 逻辑保留但视觉重写
P1：起盘结果页白色工作台
P1：合盘页白色双表单
P2：入口页统一
P2：移动端细节
```

## 16. 不做的事情

本轮不做：

- 不接线上账号系统。
- 不接支付系统。
- 不接线上私有 AI 服务。
- 不重写排盘算法。
- 不改变 release 数据结构。

本轮只修前端产品体验和本地可用逻辑。
